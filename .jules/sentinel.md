## 2024-05-31 - [Critical] Path Traversal via os.path.join in Python
**Vulnerability:** Path Traversal via os.path.join with attacker-controlled absolute paths.
**Learning:** In Python, if the second argument to `os.path.join` is an absolute path (e.g., `/etc/passwd`), `os.path.join` ignores the base path (the first argument) and returns the absolute path directly. When user input, like a file upload's `filename`, is used as the second argument, an attacker can specify an absolute path to write files outside the intended directory.
**Prevention:** Always sanitize filenames from user input by extracting the basename using `os.path.basename` and stripping out directory separators (including Windows backslashes) before passing them to `os.path.join` or other file-handling functions. Example: `safe_filename = os.path.basename(file.filename.replace('\\', '/'))`.

## 2024-06-03 - [High] Unauthenticated API Routes in Next.js App Router
**Vulnerability:** API routes under `src/app/api` were publicly accessible without authentication.
**Learning:** While `middleware.ts` was configured to protect UI routes (`/admin`, `/dashboard`, etc.), the Next.js API routes (`/api/documents`, `/api/parser`, `/api/suggestions`) were not included in the middleware matcher. As a result, endpoints proxying sensitive operations (like uploading documents or approving suggestions) to the Python backend were exposed to unauthenticated users.
**Prevention:** Always explicitly protect Next.js API routes by either adding them to the middleware matcher (if all endpoints require the same auth level) or by validating sessions at the beginning of each HTTP method handler using `getServerSession(authOptions)` and returning a `401 Unauthorized` if the session is missing. Note that certain public endpoints like `/api/health` should be intentionally excluded from this check.
